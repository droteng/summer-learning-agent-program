# Social Learning

## Principle

Social learning should be parent-first and child-friendly. Children can request connection, collaborate, and celebrate, but parents control enrollment, visibility, live sessions, and external communication.

## Friend Invitation Flow

1. Child requests to invite a friend.
2. Parent approves or rejects the request.
3. Parent sends the invite link to the friend's parent.
4. Friend's parent enrolls the child and chooses visibility settings.
5. Children can join a small Learning Squad.

## Learning Squads

Recommended squad size: 2-6 children.

Allowed by default:

- Completed mission visibility.
- Badge visibility.
- Shared project status.
- Parent-visible group comments.

Hidden by default:

- Exact scores.
- Health answers.
- Faith reflections.
- Private writing unless selected.
- Friend messages outside the squad.

## Learning Squad Room MVP

The app now supports an internal shared project room before external integrations are enabled. The room includes:

- Approved squad members and safe progress visibility.
- A shared weekly project prompt.
- Suggested collaboration roles and milestones.
- Parent-visible, moderated project updates.
- Shared reward ideas that still require parent approval.
- Parent controls for external tools, live sessions, screen share, and visibility.

Private child-to-child DMs remain disabled. Discord, Microsoft Teams, Zoom, and screen share are planned as parent-approved integrations after the internal room model is stable.

## Collaboration Projects

Project examples:

- Build a simple game together.
- Write a group story.
- Create a current-events explainer.
- Design a budget for a pretend business.
- Build a healthy family challenge.
- Create a history timeline.

## External Integrations

Start with internal parent-visible collaboration. Add external integrations later:

- Microsoft Teams for Education.
- Zoom for supervised live sessions.
- Discord only with careful age, parent, and safety controls.
- Screen share only for parent-approved sessions.
