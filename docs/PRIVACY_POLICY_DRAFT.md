# Privacy Policy Draft

This is a planning draft, not final legal text.

## Product Privacy Position

Paperclip Academy is designed as a parent-supervised learning program for children. The product should collect only the information needed to run lessons, track progress, approve rewards, support parent-controlled collaboration, and share parent-approved reports.

## Parent Control

Parents should be able to:

- Create and manage child profiles.
- Choose optional subjects and faith settings.
- Approve rewards.
- Approve friend invitations.
- Control learning squad visibility.
- Enable or disable live sessions and external integrations.
- Approve teacher/school sharing.
- Request export or deletion of child data.

## Child Data To Minimize

Avoid collecting unnecessary child data.

Required or useful data:

- First name or nickname.
- Grade level.
- Learning interests.
- Parent-selected enrichment tracks.
- Lesson progress.
- Project artifacts.
- Reward progress.
- Parent approvals.

Sensitive data to avoid or keep highly restricted:

- Home address.
- Phone number.
- Personal email for the child.
- Exact location.
- Health diagnoses.
- Private faith reflections.
- Unmoderated child messages.

## Social Data

Friend invitations should be parent-mediated. Children may request a friend invitation, but a parent sends the invitation link to the friend's parent.

Default social visibility should show:

- Completed missions.
- Badges.
- Shared project status.

Default social visibility should hide:

- Exact scores.
- Health check answers.
- Faith reflections.
- Friend messages outside the squad.
- Parent reward settings.

## Health Content

Health & Wellness content should be educational and age-appropriate. It should teach general concepts about food, movement, hydration, sleep, stress, and chronic disease prevention. It should not diagnose, treat, or shame a child.

## Faith Content

Christian Leadership must be optional and parent-controlled.

Recommended parent settings:

- No faith content.
- Christian character only.
- Christian Bible-based leadership.

## Data Retention

MVP should support:

- Parent data export.
- Parent deletion request.
- Removal from learning squads.
- Expiration of invite links.

## Pre-Launch Legal Review

Before launch, review:

- COPPA compliance.
- State privacy laws.
- Payment and subscription terms.
- School sharing and FERPA-adjacent handling.
- App store child-safety requirements.
- External messaging/live-session terms.
