# Product Spec

## Working Name

Dr. Spark Academy

## One-Line Promise

A personalized summer learning camp that keeps kids academically sharp, builds real-world skills, and turns completed learning into parent-approved rewards.

## Product Shape

Dr. Spark Academy should feel like a summer camp to the child and a reliable learning operating system to the parent.

The system uses a Principal Agent as the head coordinator. The Principal Agent assigns teacher agents, adjusts pacing, watches motivation, coordinates projects, and asks parents to approve rewards and sharing.

## Users

### Child

- Completes daily missions.
- Earns XP, mastery stars, and camp coins.
- Works on solo and parent-approved friend projects.
- Receives encouragement from a coach agent.

### Parent

- Chooses grade level, intensity, optional subjects, reward rules, faith preferences, and sharing permissions.
- Receives summaries.
- Approves friend invites, live sessions, external sharing, and rewards.

### Teacher / School Contact

- Receives parent-approved reports, transcripts, certificates, portfolios, and selected project samples.

## Core Workflow

1. Parent creates family account.
2. Parent enrolls child.
3. Child completes a short diagnostic quest.
4. Principal Agent generates an 8-week Grade 6 summer plan.
5. Parent selects optional enrichment tracks.
6. Child completes daily missions.
7. Teacher Agents adapt instruction based on performance.
8. Parent Liaison Agent sends weekly reports and reward requests.
9. Optional friend squads collaborate on projects.
10. Parent can share progress with school teachers.

## Reward System

Reward currency:

- XP: earned for effort and completion.
- Mastery Stars: earned for demonstrated understanding.
- Camp Coins: spent on parent-approved rewards.

Reward examples:

- Extra game or device time.
- Movie night.
- Fast-food or ice cream trip.
- Park, sports, swimming, or bike activity.
- Choose dinner.
- Invite a friend.
- Project showcase night.
- Small allowance bonus.
- Certificate or badge.

Rewards should encourage effort, mastery, consistency, creativity, and healthy activity.

## Social Learning

Friend invitations must be parent-approved.

Recommended flow:

1. Child requests to invite a friend.
2. Parent approves.
3. Parent sends invitation link to the friend's parent.
4. Friend's parent enrolls and approves visibility.
5. Children can join a small Learning Squad.

Learning Squads:

- 2-6 children.
- Parent-approved.
- Same grade or compatible level.
- Shared project themes.
- Limited progress visibility.
- No private unsupervised child DMs in the first version.

## Integrations

Initial:

- Email sharing for reports.
- Downloadable PDF reports and certificates.
- Private teacher share links.

Later:

- GitHub for coding projects.
- Google Classroom.
- Microsoft Teams for Education.
- Zoom for supervised sessions.
- Discord only with careful age and parent-control handling.
